# 身心障礙全流程整合導航平台 - 評估架構說明文件

本文件詳細列出平台目前使用的所有評估問題、評分機制、以及各問題間的邏輯關係。

## 1. 評估流程概覽 (UI Flow)

評估表單分為 **9 個步驟**，涵蓋起點引導、基本背景、六大功能維度、以及居住環境。

```mermaid
graph TD
    Start((開始評估)) --> S0[步驟 0: 了解您的需求 (入口分流)]
    S0 --> S1[步驟 1: 基本資料與制度狀態]
    S1 --> S2[步驟 2: 身體狀況評估]
    S2 --> S3[步驟 3: 日常生活自理能力]
    S3 --> S4[步驟 4: 認知、溝通與數位能力]
    S4 --> S5[步驟 5: 工作與經濟能力]
    S5 --> S6[步驟 6: 家庭支持情況]
    S6 --> S7[步驟 7: 心情與社交生活]
    S7 --> S8[步驟 8: 居住環境與交通方便性]
    S8 --> End((生成深度整合報告))

    subgraph "核心引導邏輯"
        S0 -- "身份/階段/痛點" --> S8
        S5 -- "不考慮工作" --> Skip[跳過細節工作評量]
    end
```

## 2. 詳細問題清單與管理機制

### 起點引導與基本背景 (步驟 0-1)
| 步驟 | 欄位名稱 | 前台題目 (白話化) | 選項 / 說明 | 影響 |
| :--- | :--- | :--- | :--- | :--- |
| **0** | `respondentType` | **1. 您目前填寫的身份是？** | 本人、家屬、其他協助者 | 調整報告語氣 |
| **0** | `currentStage` | **2. 目前大概在哪個階段？** | 住院中 ~ 規劃重返社會 | 決定導航起點 |
| **0** | `primaryConcern` | **3. 您現在最想先解決什麼？** | 我不知道回家後要怎麼照顧... | **優先建議排序** |
| **1** | `age` | 年齡 | 數值 | 報告描述 |
| **1** | `city` | 居住縣市 | 台灣各縣市 | 決定地方資源中心 |
| **1** | `education` | 教育程度 | 不識字 ~ 研究所 | 職業重建評估 |
| **1** | `hasCertificate` | 身障證明狀態 | 無、知道但未辦、未通過、已持證 | **觸發鑑定申請建議** |
| **1** | `travelCardStatus` | 愛心悠遊卡狀態 | 未辦理、辦理中、已領取 | **觸發悠遊卡申請建議** |

### 六大功能維度評估 (步驟 2-7)
評分標準統一為：**0: 較差 ~ 5: 良好** (分數越高越好、越穩定、越獨立)

#### ② 身體狀況 (Physical)
- 在家裡的移動情形
- 外出的移動情形
- 從床到椅子/馬桶的移位 (**影響輔具建議**)
- 平衡穩定度 (是否容易跌倒) (**影響跌倒風險**)
- 耐力 (走或站的持久度)
- 如果沒有人幫忙，你能不能知道或管理自己的血壓、血糖等狀況？
- **疼痛控制穩定度** (0:極痛, 5:無痛) (**影響跌倒風險**)
- 復健或運動的規律性

#### ③ 日常生活自理能力 (Self-Care)
- 自行進食、自行洗澡、自行上廁所、自行穿脫衣服
- 按時吃藥與藥物管理
- 處理家事或準備餐點
- 出門處理日常事務 (如：買東西、繳費)

#### ④ 認知、溝通與數位能力 (Cognition)
- **認知與情緒**：記憶力、定向感、日常生活判斷力、情緒行為穩定度
- **溝通與數位**：表達需求、理解他人、手機使用、電腦使用
- **AI 使用意願**：是否願意使用語音助理、ChatGPT 等工具協助生活

#### ⑤ 工作與經濟能力 (Economy)
- **前置引導**：您目前是否考慮重新工作或參與社會活動？ (**若選否則跳過細節**)
- **工作評估**：技能經驗、重返意願、體力負荷、遠距可行性、**職務支持需求**
- **經濟狀況**：目前生活費主要是靠自己還是他人？、**經濟可承受度**、目前是否有領取政府補助？

#### ⑥ 家庭支持情況 (Family)
- 家人協助意願、照顧者健康狀況
- 可提供照顧時數 (影響 **照顧崩潰風險**)
- 家人是否支持尋求外部幫助？、資源運用知識
- **家庭互動穩定度** (0:嚴重衝突, 5:無) (影響 **照顧崩潰風險**)
- 緊急備援方案 (若主要照顧者突然不能幫忙)

#### ⑦ 心情與社交生活 (Social)
- 社交互動頻率、社區參與意願
- 你平常會不會很少和人說話或互動？ (影響 **孤立風險**)
- 生活目標明確度、自我效能感 (覺得自己有能力讓生活變好)
- 睡眠品質、**情緒穩定程度** (0:嚴重困擾, 5:無) (影響 **心理支持建議**)

### ⑧ 居住環境與交通方便性 (步驟 8)
- **住家型態**：是否有電梯 (影響 **環境改善建議**)
- **主觀感受**：你會不會因為家裡或外出環境不方便，而減少活動？ (**影響穩定度分析**)
- **居家無障礙**：浴廁安全、無門檻、走道寬敞、扶手、智慧家居
- **室外便利性**：斜坡、社區電梯、人行道、**進出門口困難 (警告項)**
- **輔具與交通**：輔具類型、大眾運輸工具、**你會不會因為交通不方便，而減少出門？ (影響穩定度分析)**

## 3. 評分與分析邏輯 (Engine Logic)

### A. 維度評分 (Dimension Scores)
每個維度的分數為該類別下所有問題的 **平均值** (0-5 分)。

### B. 風險評估 (Risk Assessment)
**原則：分數越高越好。** 風險判斷統一為 `分數 <= 2` 觸發。

| 風險類型 | 觸發條件 (範例) | 說明 |
| :--- | :--- | :--- |
| **跌倒風險** | `平衡穩定 <= 2` 或 `疼痛控制 <= 2` | 建議加強居家無障礙與防滑。 |
| **照顧崩潰** | `照顧時數 <= 1` 或 `家庭互動穩定 <= 2` | 建議優先導入喘息服務。 |
| **社會孤立** | `社會互動 <= 2` 或 `社區參與 <= 2` | 長期可能導致憂鬱或功能退化。 |
| **機構化風險** | `照顧者健康 <= 2` 且 `洗澡自理 <= 2` | 居家照顧難以維持，需考慮轉銜。 |
| **經濟風險** | `經濟可承受度 <= 2` | 負擔沉重，應優先確認補助資格。 |

### C. 四象限分析 (Four-Quadrant Analysis)
用於決定「推薦路徑」。
- **穩定度 (Stability)** = (家庭支持 + 自理能力 + 身體功能) / 3
    - *修正權重*：若交通便利度 <= 2 或 居住環境進出困難，穩定度分數再乘 0.9。
- **發展度 (Development)** = (工作意願 + 工作能力 + 經濟可承受度) / 3

| 象限 | 名稱 | 核心建議 |
| :--- | :--- | :--- |
| **I** | 自立發展型 | 優先導入職業重建與社會對接 |
| **II** | 穩定生活型 | 強化生活品質與深度社區參與 |
| **III** | 潛力待援型 | 先穩固生活支持(長照)，再導入職重 |
| **IV** | 高危支持型 | 優先導入長照與照顧管理，避免崩潰 |

## 4. 關聯性與觸發動作 (Action Steps Mapping)

1.  **鑑定申請**：若 `hasCertificate` 不是 "yes"。
2.  **1966 長照**：若 `身體功能` 或 `生活自理` 平均分 <= 3。
3.  **職業重建**：若屬於第 I 或 III 象限。
4.  **環境改善**：若 `浴廁無障礙` 為 false 或住家為 `公寓/透天`。
5.  **交通卡申請**：若 `travelCardStatus` 為 "none"。
6.  **交通接送**：若交通便利評分 <= 2。
7.  **專業輔具/復健**：若身體功能評分低或移位困難。 (**新增**)
8.  **心理支持/社交**：若情緒穩定度低或生活目標感低。 (**新增**)
9. **有愛店家**：若社區參與度低或屬於第 II/IV 象限。
